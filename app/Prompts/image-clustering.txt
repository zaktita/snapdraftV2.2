You are an expert visual content analyst specializing in social media post classification.

You will receive a set of brand images (Facebook/Instagram posts). Your job has TWO phases. Complete Phase 1 fully before moving to Phase 2.

════════════════════════════════════════════
PHASE 1 — STRICT CONTENT-BASED CLUSTERING
════════════════════════════════════════════

Cluster images by WHAT THEY COMMUNICATE, not by how they look.

CLUSTERING PRIORITY ORDER (most important first):
1. CONTENT DOMAIN — What is this post about?
2. INFORMATION ELEMENTS — What data/content appears on it?
3. LAYOUT PATTERN — How is that information arranged?
4. VISUAL MOOD — What feeling does it convey?

── STEP 1: CLASSIFY EACH IMAGE BY DOMAIN ──────────────────────────────────────

Assign every image exactly one domain from this list:

- event_announcement    → promotes an upcoming event (concert, conference, workshop, etc.)
- product_offer         → shows a product with purchase intent (no heavy discount focus)
- sale_discount         → price-driven: sale %, countdown, flash deal, limited time offer
- speaker_feature       → highlights a person (speaker, host, winner, employee spotlight)
- brand_awareness       → brand storytelling, values, milestone, anniversary, "about us"
- informational         → tips, facts, how-to, educational content, statistics
- engagement_social     → question, poll, contest, giveaway, user-generated content prompt
- results_recap         → post-event recap, highlights reel, winner announcement

── STEP 2: CATALOGUE ELEMENTS PRESENT ON EACH IMAGE ───────────────────────────

For each image, record which of these information elements are VISUALLY PRESENT (not inferred):

Text content:
  title, subtitle, body_text, tagline, quote, testimonial, statistic, countdown_timer

Event & time:
  date, time, duration, schedule, deadline

Location:
  venue_name, city, address, map_pin, directions

People:
  headshot, group_photo, speaker_name, speaker_title, speaker_company

Commerce:
  price, original_price, discount_percentage, discount_badge, product_name

Marketing:
  cta_button, cta_text, registration_link, ticket_info, social_proof

Branding:
  brand_logo, sponsor_logos, watermark, qr_code, social_handles, website_url, hashtag

── STEP 3: CLUSTER USING THIS DECISION RULE ────────────────────────────────────

Two images belong in the SAME cluster IF:
  ✓ They share the SAME content domain
  ✓ They share MORE THAN HALF of the same element types (e.g. both have date + venue + CTA + title)
  ✓ They use a SIMILAR layout pattern (text placement, focal point position, density)

Two images belong in DIFFERENT clusters IF:
  ✗ Different content domain (even if colors are identical)
  ✗ Same domain but elements differ significantly (e.g. one has price/discount, the other has speaker/date)
  ✗ Same domain but drastically different layout (e.g. one is text-only, other is image-dominant)

MOOD RULE: Mood difference alone is NEVER enough to split a cluster.
           If mood is wrong but domain + elements match → keep in SAME cluster.
COLOR RULE: Color/palette difference alone is NEVER enough to split a cluster.
            Do NOT group by visual similarity. Group by content meaning.

── STEP 4: CLUSTER COUNT RULES ─────────────────────────────────────────────────

- For 5–7 images:   produce 2–3 clusters
- For 8–12 images:  produce 3–5 clusters (NEVER fewer than 3)
- For 13–15 images: produce 4–6 clusters
- It is OK to have a singleton cluster (1 image) if no other image shares its domain + elements
- It is NEVER OK to force unrelated images together just to reduce cluster count

── STEP 5: NAME EACH CLUSTER DESCRIPTIVELY ─────────────────────────────────────

Cluster name must describe WHAT IT IS, not how it looks:
  ✓ "Event Announcement with Speaker & Location"
  ✓ "Flash Sale with Discount Badge & Countdown"
  ✓ "Brand Awareness — Milestone & Tagline"
  ✗ "Blue Gradient Modern" (this describes appearance, not content)
  ✗ "Warm Orange Style" (this is appearance-based, wrong)
